# fraud_detection
# Обнаружение мошеннических транзакций

файл report.md - это краткий отчёт о проекте 

Файл requirements.txt - это файл со списком всех библиотек, необходимых для проекта

## О проекте

Проект по машинному обучению для классификации банковских транзакций как мошеннических или легитимных. Задача решается с помощью четырех алгоритмов машинного обучения с применением SMOTE для борьбы с дисбалансом классов. Датасет содержит 284807 транзакций, из которых только 492 (0.17%) являются мошенническими.

## Что делает проект

- Загружает и анализирует данные
- Применяет SMOTE для балансировки классов
- Обучает 4 модели машинного обучения
- Сравнивает модели и выбирает лучшую
- Демонстрирует работу модели на новых данных

## Установка и запуск проекта

### 1. Получение кода проекта

Склонируйте репозиторий с помощью Git или скачайте проект в виде ZIP-архива.

### 2. Установка зависимостей

Выполните команду:

pip install -r requirements.txt

Эта команда установит все необходимые библиотеки с точными версиями, указанными в файле requirements.txt. Процесс установки может занять 2-5 минут в зависимости от скорости интернета.

### 3. Скачивание датасета

Датасет не включен в репозиторий из-за большого размера. Скачайте его с Kaggle по ссылке: https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud

После скачивания распакуйте архив и скопируйте файл creditcard.csv в папку data/ вашего проекта.

### 4. Выполнение кода

В Jupyter Notebook нажмите «Выполнить все». Дождитесь завершения выполнения всех ячеек. В конце ноутбука будут показаны результаты работы моделей и демонстрация предсказаний на новых данных.

Если вы хотите выполнять ячейки поочередно, используйте Shift + Enter для выполнения текущей ячейки и перехода к следующей.

## Описание файлов проекта

- README.md - инструкция по установке и запуску проекта (этот файл)
- requirements.txt - список всех зависимостей с точными версиями
- report.md - краткий отчет с результатами и выводами по проекту
- .gitignore - список файлов и папок, исключаемых из системы контроля версий
- data/creditcard.csv - датасет, который скачивается отдельно
- models/best_model.joblib - сохраненная лучшая модель, создается автоматически при выполнении ноутбука
- mlruns/ - папка с результатами MLflow экспериментов

## Содержимое requirements.txt

Файл requirements.txt содержит следующие зависимости:

- numpy версии 1.24.3 - для работы с массивами и математическими операциями
- pandas версии 2.0.3 - для работы с табличными данными и DataFrame
- scipy версии 1.10.1 - для научных вычислений
- matplotlib версии 3.7.2 - для построения графиков
- seaborn версии 0.12.2 - для улучшенной визуализации данных
- scikit-learn версии 1.3.0 - для алгоритмов машинного обучения
- imbalanced-learn версии 0.11.0 - для применения SMOTE при борьбе с дисбалансом классов
- mlflow версии 2.5.0 - для отслеживания экспериментов и сохранения моделей
- joblib версии 1.3.1 - для сохранения и загрузки моделей
- jupyter версии 1.0.0 - для запуска Jupyter Notebook

## Результаты проекта

Лучшей моделью признан Random Forest со следующими метриками:

- F1-score равен 0.8103
- Precision равен 0.8919
- Recall равен 0.7424
- ROC-AUC равен 0.9789

Модель находит 74 процентов всех мошеннических транзакций с точностью 89 процентов.
